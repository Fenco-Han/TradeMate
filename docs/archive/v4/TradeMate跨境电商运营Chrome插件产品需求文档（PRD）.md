## 1. 文档综述

### 1.1 文档目的

本文档明确TradeMate Chrome插件的产品定位、功能架构、业务规则、交互逻辑、权限体系与非功能需求，为产品设计、开发、测试、运营全流程提供标准化执行依据，确保产品落地完全匹配跨境电商卖家前台运营全场景需求，同时实现与头部竞品的能力对标及差异化优势打造。

### 1.2 适用范围

本PRD覆盖TradeMate Chrome插件V4.0版本需求，聚焦亚马逊场景下的AI Agent自动驾驶闭环，包含插件窗口、页面植入、AI Agent决策与自动执行、合规风控、利润与现金流管控、双端数据联动与低代码工作流，不包含Web SaaS后台管理系统的界面级详细原型。

### 1.3 读者对象

产品经理、UI/UX设计师、前后端开发工程师、测试工程师、跨境电商业务负责人、市场推广人员、项目管理人员。

## 2. 产品规划

### 2.1 产品定位

TradeMate定位为**基于AI Agent + OpenClaw的亚马逊运营自动驾驶插件**。  
产品不再停留在“数据查询工具”，而是提供“情报洞察 -> 策略生成 -> 自动执行 -> 结果回传”的闭环能力，重点解决卖家在2026年面临的效率、合规、现金流和跨平台执行问题。

### 2.2 产品核心目标

#### 第一阶段目标（MVP，1-2个月）

1. 上线3个P0闭环能力：竞品实时监控与自动调价、广告智能优化与自动执行、全链路利润实时核算与异常预警。  
2. 验证核心价值：将运营手工操作时长降低40%以上，MVP试点商家周留存率达到45%以上。  
3. 建立可控执行机制：所有自动任务支持策略审批、回滚和执行审计，关键任务误执行率低于0.5%。

#### 第二阶段目标（标准版，3-4个月）

1. 完成六大能力矩阵中的核心P1能力：站外情报、多角色Agent、库存管理Agent、合规风控Agent。  
2. 新品决策周期由“周级”缩短至“天级”，广告目标达成率（如ACOS阈值控制）达到70%以上。  
3. 单人可管理SKU量提升3-5倍，形成可规模复制的运营方法论。

#### 第三阶段目标（生态版，5-12个月）

1. 上线技能市场、开放API、低代码工作流，支持企业卖家多店铺集群化运营。  
2. 覆盖多平台协同（亚马逊优先，扩展至独立站/TikTok Shop等）。  
3. 形成“自动化执行能力+合规规则引擎+生态扩展能力”的长期壁垒。

### 2.3 产品核心价值

#### 用户价值

1. **效率提升**：高频重复动作自动化，目标将人工操作时长降低50%以上。  
2. **决策升级**：从“看数据”升级为“给策略+可执行动作”，减少经验依赖。  
3. **风险可控**：合规规则实时同步，自动审计与风险预警降低封号与罚款风险。  
4. **利润可见**：全链路成本实时核算与现金流预测，避免“卖得多亏得多”。

#### 商业价值

1. **差异化定价基础**：从工具订阅转向“自动化运营价值”收费，提高ARPU与续费率。  
2. **高粘性闭环**：用户将核心执行链路托管在平台，迁移成本显著提升。  
3. **生态扩张能力**：通过技能市场与API形成平台网络效应，不依赖单一功能竞争。

### 2.4 目标用户画像

|用户类型|核心特征|核心需求|
|---|---|---|
|新手卖家（0-3个月）|预算有限、流程不熟、试错承受能力低|保姆式引导、自动合规检查、可执行的新手方案|
|成长型卖家（3-12个月）|有稳定出单、SKU数量10-50，关注人效与ROI|广告自动优化、竞品自动监控、库存与利润联动管理|
|规模化大卖（1年以上）|多店铺多站点运营、强调风控与系统打通|多Agent协同、集群化运营、可定制工作流、API与私有化能力|
### 2.5 用户核心痛点分析

|核心矛盾|典型表现|V4.0解决方向|
|---|---|---|
|效率矛盾|运营每天大量时间投入机械执行与报表整理|自动执行引擎承接重复动作，仅保留人工审批与关键决策|
|数据矛盾|站内数据失真，站外信息缺失，工具间数据冲突|站内+站外多源交叉验证，建立可信情报中枢|
|决策矛盾|数据多但无法转化为策略，执行链路断裂|多角色Agent输出策略并直接下发执行任务|
|合规矛盾|规则高频变化，人工跟踪滞后，违规成本高|规则实时同步、全流程审计、分级预警与自动整改建议|
|现金流矛盾|利润核算滞后，库存与广告投入失衡|利润实时核算+现金流预测+预算约束执行|

## 3. 整体产品架构

### 3.1 产品功能架构

采用“六大能力矩阵 + 闭环执行链路”设计，核心如下：

```Plain Text

TradeMate V4.0产品能力架构
├─ 情报洞察中枢（P0）
│  ├─ 亚马逊站内数据抓取
│  ├─ 站外平台情报采集
│  ├─ 多源交叉验证
│  └─ 蓝海机会识别
├─ 多角色AI Agent决策层（P0-P1）
│  ├─ 选品分析Agent
│  ├─ 广告优化Agent
│  ├─ 库存管理Agent
│  ├─ 合规风控Agent
│  └─ 经营分析Agent
├─ 自动化执行引擎（P0）
│  ├─ 竞品监控与自动调价
│  ├─ 广告自动执行
│  ├─ Listing自动优化
│  └─ 订单/库存自动化处理
├─ 合规风控体系（P0）
│  ├─ 规则订阅与变更解读
│  ├─ 全流程合规审计
│  └─ 风险预警与整改建议
├─ 现金流与利润管控（P0）
│  ├─ 实时利润核算
│  ├─ 现金流预测
│  └─ 成本优化与定价模拟
└─ 低代码开放平台（P1-P2）
   ├─ 可视化工作流编排
   ├─ 技能市场
   └─ 开放API与企业集成
```

### 3.2 核心业务流程

#### 3.2.1 自动驾驶总流程（核心）

目标输入（如“ACOS控制在25%以内并最大化销售额”） -> Agent拆解任务与约束 -> 多源数据拉取与校验 -> 生成策略并请求审批（可配置自动放行） -> OpenClaw执行动作 -> 回收执行结果与业务指标 -> 策略迭代。

#### 3.2.2 情报驱动选品流程

关键词/类目输入 -> 站内数据抓取 -> 站外需求与舆情抓取 -> 机会评分与风险评分 -> Agent输出选品可行性报告 -> 一键生成后续上架与投放任务。

#### 3.2.3 执行与风控流程

执行任务生成 -> 合规规则预检 -> 分环境执行（仿真/生产） -> 异常熔断与人工接管 -> 审计留痕 -> 复盘报告。

### 4.1 全局基础功能模块（P0）

#### 4.1.1 插件主窗口与全局导航

**需求描述**：插件激活后，以独立悬浮窗口形式展示，支持全局窗口控制，不影响亚马逊原生页面操作，内置全功能核心导航，实现功能快速切换。

**核心功能点**：

1. **窗口控制**：支持最小化、最大化/还原、关闭窗口，窗口可在浏览器内自由拖拽移动，最小化后收缩为侧边悬浮球；关闭窗口仅隐藏界面，后台监控与预警功能持续运行。

2. **顶部全局导航**：固定展示核心功能入口，包含**产品查询、关键词反查、关键词挖掘、关键词收录、筛查相关性、产品库、API接口**，点击可快速切换对应功能模块，切换时保留上一模块的操作记录与筛选条件。

3. **全局搜索**：导航栏右侧固定ASIN/关键词搜索框，支持切换ASIN/关键词搜索类型，输入内容后一键查询对应数据，自动带入对应功能模块。

4. **辅助功能**：导航栏内置在线咨询入口、多语言切换入口（中文/英文）、版本号展示、帮助教程入口。

**业务规则**：

- 插件窗口默认在浏览器右侧展示，支持用户自定义默认位置与窗口大小。

- 功能模块展示与全局设置开关联动，关闭的功能不在导航栏展示。

#### 4.1.2 侧边悬浮快捷入口

**需求描述**：在亚马逊所有前台页面的侧边，固定展示可拖拽的插件悬浮球，点击展开核心功能快捷入口，无需打开主窗口即可快速访问核心功能。

**核心功能点**：

1. 核心快捷入口：产品查询、关键词反查、关键词挖掘、关键词收录、筛查相关性、产品库、TikTok达人。

2. 辅助功能：入口顶部内置帮助按钮、设置按钮，支持用户快速调整插件基础配置。

**业务规则**：

- 悬浮球支持用户手动隐藏/显示，可自由拖拽至页面任意侧边位置，不遮挡亚马逊原生核心操作区域。

- 点击快捷入口，直接打开插件主窗口并跳转至对应功能模块，自动带入当前页面的ASIN/关键词信息。

#### 4.1.3 账号与权限管理体系

**需求描述**：插件内置完整账号体系，支持账号全生命周期管理，基于会员等级实现精细化权限控制，与PC端SaaS系统权限完全互通。

**会员等级与权限划分**：

|会员等级|核心权限|
|---|---|
|免费版|基础数据查看、单日有限次查询、基础功能使用、单条ASIN/关键词查询、单次最多10条批量操作、近3个月基础趋势数据查看|
|基础版|全量基础数据查看、提升单日查询上限、解锁全部基础功能、单次最多50条批量操作、基础AI功能、近1年趋势数据查看、10个主流站点数据查看|
|专业版|全量数据查看、无单日查询上限、解锁全部高级功能、无批量操作上限、全量AI功能、多店铺绑定、上架至今全量历史数据、全站点数据查看|
|企业版|专业版全量权限、子账号管理、API接口无限制调用、专属客服、定制化功能支持、私有化部署支持|
|**核心功能点**：||
1. 账号管理：支持手机号/邮箱注册、登录、密码找回、账号信息修改、账号安全设置、账号退出。

2. 会员管理：支持会员套餐查看、在线开通、续费、升级、降级，会员到期自动降级为免费版，保留用户历史数据。

3. 权限控制：严格按照会员等级控制功能使用权限、数据查看范围、操作次数上限，防止越权访问。

**业务规则**：

- 未登录账号仅开放免费版基础功能，付费功能点击后引导用户登录并开通对应会员。

- 账号权限与PC端SaaS系统完全互通，同一账号可在插件与SaaS端同步使用，数据实时同步。

#### 4.1.4 插件全局个性化设置

## 5. 非功能需求

### 5.1 性能需求

1. 插件主窗口打开响应时间≤1秒，功能模块切换响应时间≤0.3秒，无页面刷新、无数据重复加载。

2. 商品数据、关键词数据查询响应时间：单条查询≤1秒，批量100条以内查询≤3秒，全站点数据查询响应时间≤3秒。

3. 趋势图表渲染时间≤2秒，支持百万级历史数据加载无卡顿；详情页标签页切换响应时间≤0.3秒。

4. 插件运行时，浏览器内存占用≤200MB，全局功能关闭后内存占用降低至50MB以内，CPU占用率≤10%，避免造成浏览器卡顿、崩溃。

5. 批量数据导出处理时间：100条以内≤5秒，1000条以内≤15秒。

6. 页面植入模块在亚马逊页面加载完成后2秒内完成渲染，无延迟、无遮挡，不影响亚马逊原生页面加载速度。

### 5.2 兼容性需求

1. 浏览器兼容：核心适配Chrome浏览器最新5个版本，后续迭代兼容Edge、Firefox等Chromium内核主流浏览器。

2. 平台兼容：V2.0版本适配亚马逊全球17个主流站点，支持站点一键切换，数据自动适配对应站点；后续迭代覆盖Shopee、TikTok Shop等主流跨境平台。

3. 系统兼容：适配Windows、MacOS全版本操作系统，无兼容性问题。

4. 分辨率兼容：适配1080P及以上主流屏幕分辨率，界面无错乱、无拉伸、无内容遮挡，趋势图表支持响应式适配。

5. 页面兼容：与亚马逊前台页面更新兼容，页面结构调整后72小时内完成适配。

### 5.3 安全性需求

1. 账号安全：用户账号密码采用非对称加密存储，登录支持双因素认证，异地登录自动预警，防止账号被盗。

2. 数据安全：用户数据传输全程采用HTTPS加密，存储采用加密方式，严格遵守全球数据安全法规，绝不泄露用户运营数据。

3. 权限安全：严格按照用户会员等级控制功能权限，防止越权访问、功能破解，接口调用设置签名验证与频率限制。

4. 合规安全：插件开发严格遵守Chrome扩展程序开发规范、亚马逊平台用户协议，采用合规的数据获取方式，避免触发平台风控。

5. 内容安全：AI生成内容自动进行合规检测，过滤违规、侵权、虚假宣传内容，规避平台处罚风险。

### 5.4 稳定性需求

1. 插件服务全年可用率≥99.9%，计划内停机维护提前24小时通过插件弹窗、邮件通知用户。

2. 核心销售、流量、排名、搜索量数据准确率≥95%，每日更新数据源，确保数据时效性。

3. 插件运行异常时自动容错，不导致浏览器崩溃，不影响亚马逊原生页面的正常使用，异常信息自动上报。

5. 数据源多备份，避免单一数据源中断导致功能失效，确保核心功能7×24小时可用。

2. 所有指标、功能均提供名词解释，悬浮展示，降低用户理解门槛。

3. 界面设计符合亚马逊卖家使用习惯，核心操作路径不超过3步，减少用户操作成本。

4. 支持多语言切换，适配中文、英文、日文等主流语言，满足全球卖家使用需求。
