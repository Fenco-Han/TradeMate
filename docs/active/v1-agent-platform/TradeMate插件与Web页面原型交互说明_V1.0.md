# TradeMate 插件与 Web 页面原型交互说明

**文档版本**：V1.0  
**对应文档**：`docs/active/v1-agent-platform/TradeMate广告Agent功能规格说明_V1.0.md`  
**适用范围**：V1 平台底座 + 广告 Agent  
**目的**：为产品设计、前端开发、测试提供页面级原型说明

## 1. 设计原则

1. 插件承担首要触达和低摩擦处理。
2. Web 承担复杂审批、配置和复盘。
3. 所有页面优先围绕“待办”和“闭环”，而不是围绕“报表堆砌”。
4. 插件与 Web 的对象命名、状态文案必须一致。

### 2.2 Web IA

1. 登录页

3. 广告 Agent 工作台
4. 待审批页

## 3. 插件页面原型说明

### 3.1 登录态页

#### 页面目标

引导用户登录并连接正式数据。

#### 布局

```text
+----------------------------------+
| TradeMate Logo                  |
| 欢迎使用广告 Agent               |
| [邮箱/手机号输入框]               |
| [密码输入框]                     |
| [登录按钮]                       |
| [去注册] [忘记密码]               |
+----------------------------------+
```

#### 交互规则

1. 登录成功后进入插件主窗口首页。
2. 若账号未绑定可用店铺，进入空状态引导。

### 3.2 插件主窗口首页

#### 页面目标

让用户第一时间看到待处理事项和最近结果。

#### 布局

```text
+------------------------------------------------+
| 顶栏: 店铺切换 | 站点切换 | 未读提醒数          |
+------------------------------------------------+
| 今日摘要                                       |
| 待处理建议 12 | 高风险 3 | 成功任务 8 | 失败 1  |
+------------------------------------------------+
| 建议卡片列表                                   |
| [高] 暂停 Campaign X                           |
| 原因摘要...  [批准] [拒绝] [详情]              |
|------------------------------------------------|
| [中] 提高关键词竞价 Y                           |
| 原因摘要...  [批准] [拒绝] [详情]              |
+------------------------------------------------+
| 底部导航: 首页 | 任务 | 通知 | 设置            |
+------------------------------------------------+
```

#### 交互规则

1. 默认按风险等级和生成时间排序。
2. 首页只展示前 10 条待处理建议，查看更多跳转列表。
3. 点击 `批准` 时，如无需补充信息，直接发起审批或执行。
4. 点击 `详情` 打开建议详情抽屉。

### 3.3 建议详情抽屉

#### 页面目标

展示建议完整信息，支持用户做单条决策。

#### 布局

```text
+----------------------------------------------+
| 建议标题                                     |
| 风险等级: High                               |
| 目标对象: Campaign X                         |
| 推荐动作: 暂停活动                            |
|----------------------------------------------|
| 原因说明                                     |
| - 过去7天 ACOS 高于阈值 40%                  |
| - 连续5天高花费低转化                         |
|----------------------------------------------|
| 预估影响                                     |
| - 花费下降                                   |
| - 订单短期可能波动                           |
|----------------------------------------------|
| [批准] [拒绝] [稍后处理]                      |
+----------------------------------------------+
```

#### 交互规则

1. 高风险建议点击 `批准` 前需二次确认。
2. 拒绝时要求填写简短原因，可选。

### 3.4 最近任务页

#### 页面目标

查看最近执行结果和异常任务。

#### 布局

1. 任务筛选条：状态、时间、风险等级
2. 任务列表：标题、状态、执行时间、失败原因摘要
3. 点击任务进入 Web 详细页

### 3.5 通知页

#### 页面目标

集中展示消息并支持标记已读。

#### 页面状态

1. 未读状态
2. 已读状态
3. 空状态

### 3.6 设置页

#### 页面目标

管理插件显示和 Agent 行为相关配置。

#### 配置项

1. 默认店铺
2. 默认站点
3. 提醒开关
4. 展示项开关
5. 跳转 Web 默认目标页

## 4. Web 页面原型说明

### 4.1 登录页

#### 页面目标

提供正式后台入口。

#### 关键元素

1. 品牌区
2. 登录表单
3. 帮助文档入口

### 4.2 总览工作台

#### 页面目标

给用户一个统一入口查看平台待办和广告 Agent 概况。

#### 布局

```text
+--------------------------------------------------------------+
| 顶部筛选: 店铺 | 站点 | 时间区间                             |
+--------------------------------------------------------------+
| 指标卡片: 待审批 | 运行中任务 | 今日失败 | 建议采纳率         |
+--------------------------------------------------------------+
| 左: 高风险待处理建议列表    | 右: 最近任务状态                 |
+--------------------------------------------------------------+
| 下: 复盘趋势图 / 审批耗时图                                 |
+--------------------------------------------------------------+
```

### 4.3 广告 Agent 工作台

#### 页面目标

成为广告 Agent 的核心操作页。

#### 页面分区

1. 目标与约束卡片
2. 建议列表
3. 风险分布面板
4. 建议效果摘要

#### 建议列表字段

1. 建议类型
2. 目标对象
3. 风险等级
4. 原因摘要
5. 预估影响
6. 状态
7. 操作列

#### 操作列

1. 查看详情
2. 批准
3. 拒绝
4. 稍后处理

### 4.4 待审批页

#### 页面目标

承接复杂审批和批量审批。

#### 页面分区

1. 顶部筛选
2. 建议表格
3. 右侧审批详情面板

#### 批量审批交互

1. 用户勾选多条建议
2. 系统校验是否同店铺、同站点、同审批权限
3. 点击批量批准或拒绝
4. 弹出确认框
5. 返回处理结果

### 4.5 任务中心

#### 页面目标

查看所有执行任务生命周期。

#### 表格字段

1. 任务标题
2. 建议类型
3. 风险等级
4. 当前状态
5. 创建时间
6. 执行时间
7. 失败原因摘要
8. 操作

#### 任务详情页结构

1. 基本信息
2. 状态时间线
3. 关联建议
4. 关联审批
5. 审计日志
6. 跳转复盘

### 4.6 复盘分析页

#### 页面目标

展示动作前后业务效果和系统表现。

#### 页面分区

1. 任务复盘卡片
2. 指标对比图
3. 建议类型表现排行
4. 审批耗时趋势
5. 成功率趋势

#### 指标对比卡

1. 前值
2. 后值
3. 变化幅度
4. 是否达成目标

### 4.7 审计日志页

#### 页面目标

满足排查和合规追溯。

#### 筛选项

1. 操作人
2. 动作类型
3. 目标对象
4. 时间范围

### 4.8 目标配置页

#### 页面目标

为广告 Agent 提供结构化输入。

#### 表单区域

1. 目标信息
2. 风险偏好
3. 自动放行规则
4. 保存与测试按钮

#### 保存前校验

1. ACOS 在合理范围
2. 自动放行阈值必须小于高风险阈值
3. 未填必填项不能提交

## 5. 跨页面交互说明

### 5.1 插件到 Web

1. 插件点击建议详情可跳转到 Web 对应建议页。
2. 插件点击查看更多任务跳转到 Web 任务中心。
3. 插件通知点击后应打开默认浏览器标签页并定位目标页。

### 5.2 Web 到插件

1. Web 完成审批后，插件首页卡片状态实时更新。
2. Web 批量审批成功后，插件未读提醒数同步变化。

## 6. 统一页面状态

### 6.1 Loading

1. 首屏数据使用骨架屏。
2. 列表局部刷新使用行内 loading。

### 6.2 Empty

1. 无建议时提示“当前无待处理建议”。
2. 无任务时提示“暂未生成执行任务”。

### 6.3 Error

1. 网络异常提供重试。
2. 权限异常提供回到登录或切换店铺。
3. 执行异常提供查看任务详情。

## 8. 验收重点

1. 插件首页信息密度适中，可在 10 秒内完成首条建议判断。
2. Web 待审批页支持批量处理且不混淆高低风险动作。
3. 任务中心与复盘页之间跳转清晰。
4. 插件与 Web 页面状态文案一致。
