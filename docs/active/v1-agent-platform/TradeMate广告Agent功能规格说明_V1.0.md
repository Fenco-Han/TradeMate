# TradeMate 广告 Agent 功能规格说明

**文档版本**：V1.0  
**对应文档**：`docs/active/v1-agent-platform/TradeMate Agent平台产品需求文档_PRD_V1.0.md`  
**适用范围**：TradeMate 平台 V1 正式交付  
**目的**：补充到可开发、可测试、可评审的功能级规格

## 1. 文档说明

### 1.1 目标

本文件用于将 V1 广告 Agent 从平台级 PRD 细化为功能级规格，供产品、研发、测试共同使用。

### 1.2 V1 范围

V1 仅正式交付：

1. 平台共用底座最小能力
2. 广告 Agent 单闭环
3. 插件主入口与 Web 管理后台

V1 不包含：

1. 第二个正式 Agent
2. 完整站外情报模块
3. 通用工作流编排
4. 企业级组织与多租户复杂能力

### 1.3 角色定义

|角色|说明|核心权限|
|---|---|---|
|主账号|店铺负责人|目标配置、审批、执行、查看全部数据|
|操作者|广告专员|查看建议、发起审批、查看任务、查看复盘|
|审批者|被授权审批人|审批高风险建议、查看执行记录|
|只读观察者|运营负责人或老板|查看任务和复盘，不可执行|

## 3. 平台共用底座规格

### 3.1 账号与权限

#### 3.1.1 目标

实现插件与 Web 的统一账号、角色和权限控制。

#### 3.1.2 功能点

1. 邮箱或手机号登录
2. 双端共享登录态
3. 用户角色分配
4. 功能权限控制
5. 审批权限控制

#### 3.1.3 核心规则

1. 未登录用户不能访问广告 Agent 正式数据。
2. 只有主账号或授权审批者能批准高风险建议。
3. 操作者可以查看全部建议，但不可绕过审批直接执行高风险动作。

#### 3.1.4 验收标准

1. 插件与 Web 登录态 1 分钟内同步。
2. 高风险动作未授权审批时无法执行。
3. 角色变更后重新拉取权限即生效。

### 3.2 广告目标与约束配置

#### 3.2.1 目标

为广告 Agent 提供结构化目标和限制条件。

#### 3.2.2 配置项

|字段|类型|必填|说明|
|---|---|---|---|
|goal_name|string|是|目标名称|
|site|string|是|站点编码|
|store_id|string|是|店铺 ID|
|acos_target|number|否|目标 ACOS，0-100|
|daily_budget_cap|number|否|每日预算上限|
|risk_profile|enum|是|`conservative` / `balanced` / `aggressive`|
|auto_approve_enabled|boolean|是|是否启用自动放行|
|auto_approve_budget_delta_pct|number|否|预算变动自动放行阈值|
|auto_approve_bid_delta_pct|number|否|竞价变动自动放行阈值|
|status|enum|是|`active` / `paused`|

#### 3.2.3 核心规则

1. 每个店铺同一时间允许存在多个目标配置，但同一广告账户只允许一个默认生效目标。
2. 未配置目标时，系统只展示观察性建议，不允许自动放行。
3. 风险偏好决定默认阈值，但用户可覆盖配置。

#### 3.2.4 验收标准

1. 保存目标配置后，后续新建议使用最新生效配置。
2. 停用目标后，不再自动生成可执行建议，仅保留诊断信息。

### 3.3 建议中心

#### 3.3.1 目标

将广告 Agent 输出统一沉淀为可审批、可执行、可复盘的结构化建议。

#### 3.3.2 建议对象公共字段

1. 建议类型
2. 作用对象
3. 当前指标
4. 推荐动作
5. 原因说明
6. 风险等级
7. 预估影响
8. 建议状态

#### 3.3.3 建议状态

`draft -> ready -> pending_approval -> approved -> rejected -> expired -> executed`

#### 3.3.4 核心规则

1. 所有建议必须包含结构化动作和原因摘要。
2. 建议在超过有效期后自动标记为 `expired`，默认有效期 24 小时。
3. 同一对象、同一时间窗、同一动作类型的重复建议应去重或合并。

### 3.4 审批中心

#### 3.4.1 目标

对高风险建议提供人工控制机制。

#### 3.4.2 审批动作

1. 批准
2. 拒绝
3. 补充备注
4. 批量批准
5. 批量拒绝

#### 3.4.3 审批触发条件

|条件|是否必须审批|
|---|---|
|预算变动超过自动放行阈值|是|
|竞价变动超过自动放行阈值|是|
|暂停高花费活动|是|
|恢复低风险活动|可自动放行|
|连续失败后重试|是|

#### 3.4.4 核心规则

1. 批量审批仅允许同一店铺、同一站点、同一审批人处理。
2. 审批结果必须记录操作人、时间、备注。
3. 已审批建议不可重复审批。

### 3.5 任务中心

#### 3.5.1 目标

统一跟踪执行任务的生命周期。

#### 3.5.2 任务状态

`draft -> pending_approval -> approved -> queued -> running -> succeeded / failed / cancelled`

#### 3.5.3 任务操作

1. 查看详情
2. 取消未执行任务
3. 查看失败原因
4. 发起重试
5. 查看关联审计

#### 3.5.4 核心规则

1. 每条任务必须关联一条建议。
2. 每条任务必须记录 `agent_type=ad_agent`。
3. 失败任务默认最多自动重试 2 次；超过后进入人工处理。

### 3.6 通知中心

#### 3.6.1 目标

将待处理建议、审批结果和任务异常同步到插件和 Web。

#### 3.6.2 消息类型

1. 新建议生成
2. 待审批提醒
3. 审批结果通知
4. 任务成功通知
5. 任务失败通知
6. 熔断与异常升级

#### 3.6.3 核心规则

1. 插件优先展示高优先级未读消息。
2. Web 消息中心保留完整历史。
3. 同一消息在插件和 Web 的已读状态保持同步。

### 3.7 审计中心

#### 3.7.1 目标

确保所有关键动作都有追踪记录。

#### 3.7.2 需要审计的事件

1. 目标配置变更
2. 建议审批
3. 任务执行
4. 任务取消
5. 任务重试
6. 自动放行触发

#### 3.7.3 核心规则

1. 审计日志普通用户不可删除。
2. 所有审计日志必须带 `actor`、`action`、`target`、`timestamp`、`result`。

### 3.8 复盘中心

#### 3.8.1 目标

将广告 Agent 的建议和执行结果转化为可回看的业务效果。

#### 3.8.2 复盘内容

1. 动作前后核心指标对比
2. 建议采纳率
3. 执行成功率
4. 审批耗时
5. 目标达成趋势

#### 3.8.3 核心规则

1. 每个成功或失败任务都生成一条复盘快照。
2. 复盘快照默认按任务完成后 24 小时生成一次首版结果。
3. 若关键数据未齐，应标记为 `partial` 状态，不展示虚假完整结论。

## 4. 广告 Agent 规格

### 4.1 数据范围

V1 广告 Agent 覆盖以下对象：

1. Campaign
2. Ad Group
3. Keyword
4. Search Term

V1 重点展示以下指标：

1. 花费
2. 点击量
3. CTR
4. CPC
5. 订单量
6. 销售额
7. ACOS
8. ROAS
9. 转化率
10. 预算消耗率

### 4.2 建议类型矩阵

|建议类型|作用对象|触发条件摘要|默认风险等级|执行动作|
|---|---|---|---|---|
|budget_increase|Campaign|预算消耗率高且 ACOS 达标|medium|提高预算|
|budget_decrease|Campaign|预算消耗率低且 ACOS 不达标|medium|降低预算|
|bid_increase|Keyword|高转化、低曝光损失|medium|提高竞价|
|bid_decrease|Keyword|高花费、低转化|medium|降低竞价|
|campaign_pause|Campaign|连续低效且超出容忍阈值|high|暂停活动|
|campaign_resume|Campaign|历史有效且恢复条件达成|low|恢复活动|
|negative_keyword_add|Search Term|高花费零转化或低相关性|high|加入否词|

### 4.3 触发规则细化

#### 4.3.1 提高预算 `budget_increase`

触发条件：

1. 过去 3 天预算消耗率 >= 90%
2. 过去 7 天 ACOS <= 目标 ACOS
3. 订单量或销售额趋势非下降

不触发条件：

1. 当前活动已在近 24 小时内被提高预算
2. 已超过店铺预算上限

#### 4.3.2 降低预算 `budget_decrease`

触发条件：

1. 过去 3 天预算消耗率 <= 50%
2. 过去 7 天 ACOS 高于目标阈值 20% 以上

#### 4.3.3 提高竞价 `bid_increase`

触发条件：

1. 转化率高于账户中位数
2. 当前曝光份额不足
3. CPC 未超过风险上限

#### 4.3.4 降低竞价 `bid_decrease`

触发条件：

1. 花费高于阈值
2. 转化率低于目标
3. 近 7 天无明显改善

#### 4.3.5 暂停活动 `campaign_pause`

触发条件：

1. 连续 7 天花费高、转化低
2. ACOS 严重偏离目标
3. 不属于人工保护名单

#### 4.3.6 恢复活动 `campaign_resume`

触发条件：

1. 被暂停活动重新满足预算和相关性条件
2. 手工标记为可恢复

#### 4.3.7 否词 `negative_keyword_add`

触发条件：

1. Search Term 花费达到阈值
2. 订单为 0 或明显低转化
3. 相关性评分低

### 4.4 风险分级

|风险等级|定义|审批要求|
|---|---|---|
|low|对投放影响小、可逆性强|可自动放行|
|medium|对预算或竞价有中等影响|视阈值决定|
|high|可能显著影响投放表现|必须人工审批|

### 4.5 自动放行规则

仅当以下条件同时满足时允许自动放行：

1. 用户显式启用自动放行
2. 动作为 `low` 或满足 `medium` 阈值
3. 当前店铺无熔断状态
4. 同类型任务近 24 小时失败率低于 10%

### 4.6 插件交互规格

#### 4.6.1 插件首页

展示内容：

1. 今日待处理建议数
2. 高风险待审批数
3. 最近成功任务数
4. 最近失败任务数

可执行操作：

1. 查看建议详情
2. 单条批准
3. 单条拒绝
4. 标记稍后处理
5. 跳转 Web 批量处理

#### 4.6.2 建议卡片

卡片字段：

1. 建议标题
2. 目标对象
3. 推荐动作
4. 风险等级
5. 原因摘要
6. 预估影响
7. 生成时间

### 4.7 Web 交互规格

#### 4.7.1 广告 Agent 工作台

包含：

1. 目标卡片
2. 建议列表
3. 风险分布
4. 任务状态摘要

#### 4.7.2 待审批页

支持：

1. 按风险等级筛选
2. 按店铺、站点筛选
3. 批量审批
4. 查看审批历史

#### 4.7.3 任务中心

支持：

1. 查看所有任务状态
2. 查看失败原因
3. 重新执行
4. 跳转复盘

#### 4.7.4 复盘页

支持：

1. 任务级复盘
2. 建议类型级复盘
3. 时间区间筛选

## 6. 测试重点

4. 单条与批量审批流程
5. 任务重试与熔断逻辑
6. 复盘结果生成完整性
