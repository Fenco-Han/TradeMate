# TradeMate Codex任务清单与执行计划

**文档版本**：V1.0  
**适用范围**：TradeMate V1 平台底座 + 广告 Agent  
**执行方式**：默认由 Codex 在当前仓库内分阶段连续完成  
**目标**：把当前文档基线转成可直接执行的工程任务清单

## 1. 执行原则

### 1.1 总体原则

1. 先搭底座，再接广告 Agent。
2. 先打通主链路，再补 fallback。
3. 先实现 API-first，再实现 OpenClaw browser fallback。
4. 所有任务以 `docs/active/v1-agent-platform/` 下文档为唯一基线。

### 1.2 Codex 执行原则

1. 每次只处理一个明确任务包。
2. 每个任务包必须包含代码、测试、文档或至少一种验证结果。
3. 涉及 schema、接口、状态机变更时，必须同步更新对应文档。
4. 未冻结的新需求不插入正在进行的任务包。

### 1.3 完成定义

一个任务包只有同时满足以下条件才算完成：

1. 代码已提交到工作区。
2. 本地可运行或至少可编译。
3. 有最小验证结果。
4. 关键接口、配置或结构变化已写入文档。

## 2. 工作包结构

### 2.1 工作流分组

本次开发拆为 6 条主工作流：

1. 工程与基础设施
2. 后端平台底座
3. 广告 Agent 服务
4. Web 管理后台
5. Chrome 插件
6. OpenClaw 扩展与 fallback

### 2.2 阶段划分

|阶段|目标|说明|
|---|---|---|
|P0|基础工程可运行|建立仓库骨架、环境、CI、文档入口|
|P1|平台底座可用|账号、权限、目标、建议、审批、任务、通知、审计|
|P2|广告 Agent 主链路可用|规则引擎、执行器、复盘、双端联通|
|P3|OpenClaw fallback 与灰度准备|fallback、稳定性、灰度发布准备|
